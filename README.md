এই প্রজেক্টটি জাভাস্ক্রিপ্টের আধুনিক লজিক ব্যবহার করে তৈরি একটি ইস্যু ট্র্যাকিং সিস্টেম। নিচে জাভাস্ক্রিপ্টের কিছু গুরুত্বপূর্ণ কনসেপ্ট সহজ ভাষায় বুঝিয়ে বলা হলো:

১. var, let, এবং const-এর মধ্যে পার্থক্য (The Difference between var, let, and const)
জাভাস্ক্রিপ্টে ভেরিয়েবল রাখার জন্য এই তিনটির স্বভাব আলাদা:

var: এটি পুরনো আমলের পদ্ধতি। এটি Function-scoped, অর্থাৎ একটি ফাংশনের ভেতর একে ডিক্লেয়ার করলে ওই ফাংশনের সব জায়গা থেকে একে পাওয়া যায়। এর বড় সমস্যা হলো, একই নামে বারবার ভেরিয়েবল ডিক্লেয়ার করা যায়, যা পরে কোডে ঝামেলা তৈরি করে।

let: এটি আধুনিক এবং Block-scoped। এটি শুধু নির্দিষ্ট এক জোড়া সেকেন্ড ব্র্যাকেটের { } ভেতর কাজ করে। এর মান তুমি পরে চাইলে পাল্টাতে পারবে, কিন্তু একই ব্লকে একই নামে দ্বিতীয়বার ডিক্লেয়ার করা যায় না।

const: এটিও Block-scoped, কিন্তু এর মান একদম ফিক্সড বা ধ্রুবক। একবার মান সেট করে দিলে সেটা আর পাল্টানো যায় না। যেমন কারো জন্মতারিখ বা কোনো নির্দিষ্ট ডাটা যা কখনো বদলাবে না।

২. Spread operator (...) কী?
এই তিনটা ডট (...) আসলে ডাটা ছড়িয়ে দেওয়ার জাদুকরী উপায়। এটি কোনো অ্যারে (Array) বা অবজেক্ট (Object)-এর ভেতরকার সব উপাদানকে বাইরে বের করে বা ছড়িয়ে দিতে ব্যবহার হয়।

কেন ব্যবহার করবে? ধরো তোমার একটা লিস্ট আছে আর তুমি চাচ্ছো ওই লিস্টের সব কিছু কপি করে নতুন একটা লিস্ট বানাতে বা দুইটা লিস্টকে জোড়া দিতে—তখন এই Spread operator জাদুর মতো কাজ করে।

৩. map(), filter(), এবং forEach()-এর পার্থক্য
এই তিনটি মেথডই অ্যারের ওপর লুপ চালানোর জন্য ব্যবহার হয়, কিন্তু এদের কাজের উদ্দেশ্য ভিন্ন:

forEach(): এটি শুধু একটা লুপ চালায়। প্রতিটি এলিমেন্টের ওপর দিয়ে ঘুরে যায় কিন্তু নিজে থেকে নতুন কোনো অ্যারে তৈরি করে না।

map(): এটিও লুপ চালায়, কিন্তু এর স্পেশালিটি হলো এটি প্রতিটি এলিমেন্টের ওপর কাজ করে দিনশেষে তোমাকে একটা নতুন অ্যারে উপহার দেয়।

filter(): এটি হলো ছাঁকনির মতো। তুমি একটা শর্ত (Condition) দেবে, আর সে ওই শর্ত মেনে ডাটাগুলোকে ছেঁকে তোমাকে একটি নতুন অ্যারে দেবে।

৪. Arrow function কী?
আগে আমরা ফাংশন লিখতাম function name() { } এভাবে। এখন আধুনিক জাভাস্ক্রিপ্টে আরও সংক্ষেপে এবং স্টাইলিশ ভাবে ফাংশন লেখা যায়, যেটাকে বলে Arrow function: () => { }। এটি কোডকে পরিষ্কার রাখে এবং পড়তে সুবিধা হয়।

৫. Template literals কী?
আগে টেক্সটের সাথে ভেরিয়েবল জোড়া দিতে অনেক প্লাস চিহ্ন (+) ব্যবহার করতে হতো। Template literals সেই ঝামেলা দূর করেছে। ব্যাকটিক (`) ব্যবহার করে তুমি খুব সহজেই ${variable} এভাবে সরাসরি ভেরিয়েবল লিখে দিতে পারো। এতে কোড লেখা এবং বোঝা অনেক সহজ হয়ে যায়।
